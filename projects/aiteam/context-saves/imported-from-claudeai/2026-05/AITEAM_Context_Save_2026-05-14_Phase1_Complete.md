## THE BIG FINDING — OPUS-NOT-SONNET BUG

**Most important discovery of the session.** While patching `ai-do.sh` to log token usage, CC discovered the script had no `--model` flag. `claude -p` was defaulting to whatever Claude Code config said — which on this machine was `claude-opus-4-7[1m]` (Opus 4.7, 1M context).

**Implication:** Step 17's "Sonnet won at r=0.995" result was Opus-vs-Opus from two separate calls. The whole tier test was invalid on its premise. The "_sonnet_scores.csv" file was generated by Opus.

**Remediation:**
- `ai-do.sh` now explicitly pinned to `--model sonnet`
- Step 16 re-run with real Sonnet
- Real result: composite r=0.933 vs manual baseline (= Opus). Drops from 0.995 to 0.933. MAE 0.05 → 0.363.
- Decision: **Sonnet remains production judge** at r=0.933 (still in the published-empirical-judge band of 0.85-0.95). 2.7× cost savings vs Opus justifies the ~6.7% accuracy loss.
- Audit log integrity preserved with retraction (row 18) + v2 result (row 19) instead of editing the original row 17.

**Lesson logged:** Sign-off discipline catches premature ✅ on visible failures. But it can't catch silent runtime bugs. The only protection is verifying premises whenever something seems too clean (r=0.995 was suspicious — should have been a flag at the time).
